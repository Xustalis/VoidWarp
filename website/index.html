工作原理

简单三步，实现设备间的无缝连接

1. 自动发现

利用 UDP 广播技术，无需手动输入 IP 地址，设备打开应用即可在局域网内相互“看见”。

2. 安全握手

设备间交换公钥进行身份验证，建立加密通道。所有请求需经用户确认，杜绝恶意连接。

3. 极速传输

建立 TCP 直连通道，充分利用局域网带宽（可达 100MB/s+），实现大文件秒传。

无网环境传输

在飞机、地下室或网络受限的办公环境中，依然可以进行稳定的文件交换。

传输过程不经过第三方服务器，敏感文档、私人照片完全掌握在自己手中。

跨生态协作

打破 Android 与 Windows 之间的壁垒，无需微信文件助手，传文件从未如此简单。

Rust + FFI
底层统一，体验原生

我们拒绝使用 Electron 或 Web 技术构建核心。95% 的业务逻辑（网络、加密、文件 I/O）下沉在高性能的 Rust 层，通过 FFI 暴露给上层 UI。

01

Unified Core (统一核心)

一份 Rust 代码编译为 .so (Android) 和 .dll (Windows)，保证双端行为绝对一致，减少维护成本。

02

Native UI (原生界面)

使用 Kotlin (Jetpack Compose) 和 C# (WPF) 构建 UI，提供极致的启动速度和完美的系统集成（如通知、拖拽）。

常见问题

安装和使用过程中的常见疑问

传输速度有多快？
速度主要取决于您的路由器性能和网络环境。在 5GHz Wi-Fi 环境下，VoidWarp 通常可以达到 30MB/s - 100MB/s 的速度。由于我们使用 Rust 编写底层并优化了缓冲区，通常比基于 HTTP 的传输工具更快。
需要连接互联网吗？
完全不需要。VoidWarp 仅在局域网（LAN）内工作。您的数据从一台设备直接流向另一台设备，不经过任何外部服务器。这也意味着即使断网，只要路由器通电，依然可以使用。
为什么我的设备互相找不到？
请确保两台设备连接到同一个 Wi-Fi 网络。如果在 Windows 上，请检查防火墙设置，确保允许 VoidWarp 进行专用网络通信。在某些公共 Wi-Fi（如学校、酒店）中，AP 隔离功能可能会阻止设备发现。
这个项目是开源的吗？
是的，VoidWarp 是 100% 开源的。您可以访问我们的 GitHub 仓库查看源代码、提交 Issue 或贡献代码。
